# 第三方 AI 工具收藏 👁️

本项目是为个人核心 AI Agent（如 Claude Code, Cursor, Windsurf 等）构建的能力扩展收藏库。我们从互联网上搜集优秀的第三方 Skill、MCP 服务、CLI 脚本及提效工具，并**严格执行“去广告与软文净化”标准**进行归档，确保存档内容的纯净度与技术专业性。

---

## 📂 科学分类体系

本项目按功能及技术实现将 AI 工具划分为以下四大类：

1. **🌐 网页检索与数据抓取 (Web Retrieval)**
   * **定位**：解决 AI Agent 的“断网”或“信息壁垒”问题。
   * **内容**：提供网页抓取、多平台（推特、B站、Reddit 等）免 Key 阅读、全网语义搜索等工具。
   
2. **🔌 MCP 服务 (Model Context Protocol)**
   * **定位**：利用 Anthropic 推出的 Model Context Protocol，扩展 Agent 的外部连接能力。
   * **内容**：集成各种第三方 MCP 服务器，提供数据库访问、本地软件交互等服务。

3. **🧠 Agent 专属技能 (Agent Skills)**
   * **定位**：直接供 AI 代理读取并执行的指令集（如 Cursor `.cursorrules`，Gemini `SKILL.md` 等）。
   * **内容**：针对特定工作流的结构化 prompt、微操作检查清单。

4. **🛠️ 开发者提效工具 (Developer Tools)**
   * **定位**：辅助 Agent 开发、测试、监控和提效的底层基建。
   * **内容**：调试工具、本地沙盒环境配置、自动化测试脚本。

## 🧹 全局内容净化原则

为保护开发者和 Agent 的注意力，本仓库内收录的所有工具文档均经过严格的内容净化，默认剔除以下无效内容：
* **广告与推广链接一律移除**：微信号、群二维码、商业推广短链接、非必要的友情赞助链接。
* **营销话术全面清除**：如“业务合作”、“限时优惠”、“关注领资料”等。
* **内容保真与最小干预**：只删除推广信息，严格保留并优化原文的技术实现、配置参数和客观事实，不作主观改写。

## 🚀 全局使用说明

### 1. 如何为你的 Agent 导入这些工具

对于大多数现代 AI 代理客户端，导入新能力主要有以下几种方式：

* **Skill/指令文件导入**：
  将本仓库中对应工具的 `SKILL.md` 或说明内容复制到你项目根目录下的 `.agents/skills/` 或配置的 Skill 路径下。Agent 会在启动时自动读取并学会如何调用它。
* **MCP 服务注册**：
  根据对应 MCP 工具的说明，在你的客户端配置文件（如 `Claude` 客户端的 `claude_desktop_config.json` 或 Cursor 的 MCP 设置界面）中添加该 MCP 服务的启动命令。
* **系统 CLI 工具安装**：
  直接在宿主机本地环境执行对应工具的 `install` 命令（例如 `pip install` 或 `npm install`），确保 Agent 能够通过 Terminal 执行该 CLI 指令。

### 2. 诊断与环境检查
在让 Agent 使用新工具前，建议先运行各工具自带的诊断或体检命令（如 `doctor` 命令），以确保本地的网络代理、Cookie 及登录态均已就绪。
