## v3.6.0
> *2026/07/09*

### ✨ 신규

- **동기화**: 동기화 프로토콜에 슬라이딩 윈도우 파이프라인 방식을 추가. 업로드 배치 전송과 다운로드 페이지 전송이 기존에는 한 건씩 확인을 기다리는 방식(stop-and-wait)이었으나, 이제 여러 배치/페이지를 동시에 전송할 수 있어 대용량 Vault의 전체 동기화 시간이 크게 단축됨(실측 기준 업로드 6.64배, 다운로드 3.19배 향상). 전 과정이 협상(negotiation) 기반으로 양방향 하위 호환을 보장하므로, 신구 클라이언트와 서버의 어떤 조합이든 자동으로 기존 순차 방식으로 되돌아가며 강제 업그레이드가 필요 없음.
- **설정**: `pipeline-window-up`(기본값 `8`)과 `pipeline-window-down`(기본값 `4`) 두 항목을 추가. 각각 업로드/다운로드 파이프라인에서 동시에 전송 중일 수 있는 배치/페이지 수를 제어함. 명시적으로 `0`으로 설정하면 즉시 stop-and-wait 방식으로 롤백되며, 파이프라인 기능의 런타임 안전장치 역할을 함(관리자 패널에서도 읽기/쓰기 가능).

### 🚀 최적화

- **동기화**: 다운로드 동기화 청크 크기 `sync-down-chunk-num`의 기본값을 `50`에서 `200`으로 상향해 stop-and-wait 방식에서의 페이지 왕복 횟수를 줄임.
- **동기화**: 전체/증분 동기화 시 노트 본문을 필요할 때만 읽고 실제로 페이지를 전송할 때 내용을 채우도록 변경해, 대용량 Vault에서의 순차 블로킹 읽기와 전체 메모리 적재를 방지.
- **데이터베이스**: `note` 테이블에 `(vault_id, path_hash)` 복합 인덱스를 추가해, 전체 업로드 시 단건 조회가 풀 테이블 스캔으로 저하되는 문제를 해소.
- **데이터베이스**: 전문 검색 색인 기록, 수정 여부 확인, 일괄 삭제 등 경로에 있던 여러 중복 쿼리를 제거(`upsertFTS`가 호출자가 방금 기록한 레코드를 재사용, `NoteModify`의 중복 쿼리 통합, 일괄 삭제 시 건별 존재 여부 확인 제거).
- **동기화**: 쓰기 큐 워커가 대기 중인 작업을 논블로킹 방식으로 일괄 처리하도록 변경해 건별 스케줄링 오버헤드를 줄임.
- **동기화**: `sync_log` 기록 방식을 건별 단독 goroutine에서 유계 채널 + 단일 워커의 일괄 기록 방식으로 변경해, 대량 업로드 시 goroutine이 폭증하는 문제를 방지.
- **동기화**: 브로드캐스트 발송을 락 외부에서 동시에 처리하도록 변경해, 응답이 느린 기기가 동일 사용자의 다른 온라인 기기로의 메시지 전달을 지연시키지 않도록 개선.
- **성능**: 노트 diff 처리 시 기준 텍스트 길이에 따라 라인 단위 전처리를 동적으로 활성화해 대용량 노트 병합 속도를 향상.
- **성능**: 폴더 개수 집계를 폴더별 `COUNT` 2회 조회(N+1 문제)에서 단일 `GROUP BY` 집계 쿼리로 변경. 파일 경로 마이그레이션 확인, `git-sync` 활성화 여부 판단 등 자주 호출되는 경로에 프로세스 내 캐시를 도입해 중복 I/O와 쿼리를 줄임.

### 🛠️ 수정

- **동기화**: 동일 노트/파일에 대한 동시 쓰기 시 `singleflight`를 잘못 사용해, 나중에 도착한 쓰기가 성공 응답을 반환하면서도 조용히 유실되는 데이터 손실 버그를 수정.
- **동기화**: 노트/첨부파일 이름 변경 시 `os.Rename` 실패가 조용히 무시되어 데이터베이스와 디스크 상태가 불일치하던 문제를 수정.
- **안정성**: 백업, Git 동기화 등 백그라운드 단독 goroutine에 panic recover를 추가해, 단일 작업의 예외로 서버 프로세스 전체가 크래시되지 않도록 개선.
- **동기화**: 업로드 배치 프로토콜에 배치 인덱스 기반 중복 제거를 추가해, 클라이언트 재전송으로 인한 서버 측 중복 기록과 카운트 오류를 방지.
- **동기화**: 다운로드 페이지 `PageAck`이 예상과 다를 경우 서버가 현재 페이지를 재전송하는 폴백을 추가해, 클라이언트가 10분 세션 TTL까지 멈춰있는 문제를 방지.
- **동시성**: 병렬 처리 활성화 시 경쟁 상태(race condition)를 유발할 수 있던 WebSocket 클라이언트 정보 필드의 잠금 없는 읽기/쓰기를 수정.
- **안정성**: DB 커넥션 풀의 LRU 제거 로직에 최소 유휴 시간 보호를 추가해, 사용 중인 커넥션이 실수로 닫히는 문제를 방지.
- **설정**: 롤백 안전장치가 절반만 동작하던 문제를 수정——`pipeline-window-up`/`pipeline-window-down`을 명시적으로 `0`으로 설정해도 설정의 2차 기본값 채움 로직에 의해 조용히 기본값으로 덮어써지던 것을, 명시적인 `0`을 올바르게 인식하고 유지하도록 수정.
- **동기화**: 페이지네이션 봉투 필드 `pageIndex`의 0페이지가 Protobuf 모드에서 비페이지네이션 브로드캐스트 메시지와 구분되지 않던 제로값 모호성 문제를 수정.
