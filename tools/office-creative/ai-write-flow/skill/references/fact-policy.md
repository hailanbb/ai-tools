# fact-policy.md — 全局事实核验输出门禁

> 所有输出模式（longform_article / card_post）共同遵守的事实准确性底线。
> 这是**输出门禁**，不是调研过程规范——调研过程由 research-config.md 负责。

---

## 适用范围

本策略约束所有进入最终输出的事实表述，无论输出形态是：

- 公众号长文（longform_article）
- 图卡文案（card_post）
- 审校修订后的文章

---

## 高风险字段清单

以下字段必须有 Tier 1/2 官方来源支撑，才能进入任何输出：

| 字段类型 | 示例 |
|---------|------|
| 版本号 | v0.16.0、3.5 Sonnet、GPT-4o |
| 发布时间 / 日期 | 2025 年 6 月发布、昨日上线 |
| 性能数据 | 速度提升 3 倍、延迟降低 40% |
| 模型名称 | Claude 3.5 Sonnet、Gemini 1.5 Pro |
| 支持平台 / 系统 | 支持 macOS / Windows / Linux |
| 安装命令 / API 地址 | pip install hermes-agent==0.16.0 |
| 价格 / 限制 | 免费版每月 100 次、$20/月 |
| 官方集成 / 合作 | 官方支持 MCP 协议 |
| 功能名称归属 | 该功能属于 X 产品，不是 Y 产品 |
| 架构变化 | 从单体改为微服务、新增流式输出 |

## 输出门禁规则

### 禁止写入输出的事实状态

| claim 状态 | 处理方式 |
|-----------|---------|
| `supported`（官方来源核验） | 可写入输出 |
| `unverified`（找不到官方来源） | 禁止写入输出；必须删除或弱化表达 |
| `contradicted`（与官方来源矛盾） | 禁止写入输出；必须阻断并等待用户确认 |
| stale（来源超过 30 天）| 禁止写入发布内容；背景介绍须显式标注 "来源日期" |

### 弱化表达规范

当找不到官方来源，但信息对叙事有价值时，允许弱化表达：

| 禁止写法 | 弱化写法 |
|---------|---------|
| 速度提升了 180 倍 | 官方暂未提供量化数据 |
| 支持所有主流平台 | 已知支持 macOS，其他平台待官方确认 |
| 完全免费 | 当前文档未说明定价，建议以官网为准 |

---

## 禁止表述

无论哪种输出模式，以下绝对表述都被禁止：

- ❌ 「100% 准确」→ 改为「当前已由官方来源核验」
- ❌ 「史诗级」、「全网最强」、「首个」（无法核验时）
- ❌ 「唯一」、「完全免费」、「无限」（无官方明确说明时）
- ❌ 「专家认为」、「据统计」等模糊归因（须换成具体来源）

## 来源质量要求

**高风险字段必须由 Tier 1 或 Tier 2a 来源支撑**（见 `research-config.md` Layer 2）：

| Tier | 来源描述 | 可支撑高风险字段 |
|------|---------|---------------|
| 1 | 官方网站、release notes、官方文档、GitHub releases | ✅ 是 |
| 2a | 官方维护者声明（创始人/核心维护者以官方身份发布的博客/演讲/声明）| ✅ 是 |
| 2b | 第三方分析（非官方博客、媒体分析、社区转述）| ❌ 仅找线索 |
| 3/4 | 科技媒体、社区帖子、论坛 | ❌ 仅找线索 |

优先来源顺序（越靠前越可信）：

1. 官方网站 / 官方公告
2. 官方 GitHub Release / Changelog / Issues
3. 官方文档（docs.xx.com 等）
4. PyPI / npm / package registry 官方元数据
5. 官方社交账号或公开声明
6. 官方维护者在官方身份下的声明（Tier 2a）

禁止将以下来源作为高风险字段的**唯一支撑**：

- 非官方技术博客（即使博主很有名）
- 社区转述（Reddit、V2EX、知乎）
- 未注明作者的转载文章
- 截图（截图可作为候选事实起点，须经外部来源核验后才能进入输出）
