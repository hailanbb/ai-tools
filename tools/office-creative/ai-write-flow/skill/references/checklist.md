# checklist.md — 多轮审校清单

> Step 5 执行时加载。内容、作者手迹、AI 味、细节、二次扫描按顺序执行，不可跳过。

---

## 结构审校（随审校流程同步执行）

- [ ] `#` 标题在全文出现且仅出现一次
- [ ] 文中不存在 `###` 层级标题（禁止使用）
- [ ] 开头段落（`#` 后、第一个 `##` 前）未出现 `####`
- [ ] 每个普通 `##` 章节包含至少一个编号 `####` 小标题（`####` ①）
- [ ] 每个 `##` 内的 `####` 编号从 `①` 开始、连续递增（不跳号）
- [ ] 全文最后一个章节为固定结尾 `## 写在最后`（全文唯一，且是最后一个 `##`）
- [ ] `## 写在最后` 下**不得**包含 `####` 小标题
- [ ] 普通 `##` 章节的 `####` 小标题数量为 2-3 个（建议项，不强制）
- [ ] 普通 `##` 章节能独立提炼摘要（满足未来章节配图语义输入需求）

---

## 第一遍：内容审校

- [ ] 技术信息准确：版本号正确，术语拼写正确，API 调用准确
- [ ] 所有性能数据来自真实测试，测试环境明确（硬件 + 软件版本）
- [ ] 对比数据前后一致（表格数字 = 正文数字，百分比计算正确）
- [ ] 观点前后无矛盾，段落逻辑连贯，结论有事实支撑
- [ ] 代码块语法正确、可运行，包含必要的 import 和配置
- [ ] 所有案例来自真实经历，没有编造的项目数据
- [ ] 核心结论 100% 可追溯（每条结论都有来源链接或真实数据）
- [ ] 开篇 3 秒内有钩子（痛点 / 悬念 / 数据冲击）
- [ ] 结尾包含'扩展阅读'或具体的下一步行动建议

---

## 第二遍：作者手迹保护

- [ ] 已判断审校对象是“已有文章”还是“新生成文章”
- [ ] 已有文章只改真正影响理解、节奏或事实准确性的地方
- [ ] 未默认删除作者的语气词、重复、口语停顿和自我修正
- [ ] 未把作者的平直陈述改成对仗、金句或文案腔
- [ ] 未擅自添加时间、地点、情绪、数据、例子
- [ ] 关键改动已说明原因；拿不准的改动标注“可还原”

## 第三遍：AI 味检测/降噪

### P0 必删项

- [ ] 无聊天残留
  - ❌ '当然可以，很高兴为您解答…'
  - ❌ '希望这对你有帮助'
  - ✅ 直接进入正文

- [ ] 无 AI 腔开场白
  - ❌ '当然可以，很高兴为您解答…'
  - ✅ 话不多说，进入正题

- [ ] 无空洞套话
  - ❌ '在当今 AI 技术飞速发展的时代…'
  - ✅ 'AI 工具发展很快，但这个问题还没解决…'

- [ ] 无机械连接词
  - ❌ '首先…其次…最后…综上所述…'
  - ✅ 直接陈述，逻辑顺序自然体现

- [ ] 无模糊归因
  - ❌ '专家认为…研究表明…据统计…'
  - ✅ '[具体机构/作者名] 在 [文章/报告] 中指出…'

- [ ] 无夸张美学
  - ❌ '这是一次革命性的里程碑式突破'
  - ✅ '这让开发效率提升了 40%'

### P1 高频句式

- [ ] 无高频“不是 X，而是 Y / 与其说 X，不如说 Y”假深刻结构
- [ ] 无普通事实后的意义拔高
  - ❌ '这标志着开发范式被重新定义'
  - ✅ 说明具体变化和证据
- [ ] 无连续排比三连
  - ❌ '更快、更稳、更智能'
  - ✅ 只保留真实差异，必要时展开
- [ ] 无连续短句轰炸
  - ❌ '没有犹豫。没有退路。只有向前。'
  - ✅ 合并成自然句

### P2 密度信号

- [ ] '显著提升' → 具体数字
- [ ] '充分利用' → '用好'
- [ ] '进行操作' → 直接动词
- [ ] '实现功能' → 具体功能名
- [ ] '有助于' → '能'
- [ ] '使得' → '让'
- [ ] 空洞好词密度不过高：高效、强大、完善、优秀、稳定
- [ ] 软文词密度不过高：赋能、打造、沉淀、闭环、生态、抓手

### 结构强制项

- [ ] 标点风格前后一致，无明显混用
- [ ] 破折号 `——` / em dash `—` 每篇最多 1 处，且必须有必要性
- [ ] 无 emoji 装饰标题和正文
- [ ] 无机械加粗；加粗只服务扫读，不制造格式噪音

- [ ] 工具对比/功能更新场景有 Before vs Now 对比表格
  - ❌ '速度提升了很多'
  - ✅ `| 维度 | 以前 | 现在 |` 格式表格

- [ ] 每段有一个加粗关键词，形成扫读路径
  - ❌ 全段无加粗
  - ✅ 每段一处 `**核心词**`

- [ ] 四字词连续堆砌已拆解
  - ❌ '功能强大、性能卓越、操作简便'
  - ✅ '点两下就能用，跑起来比之前快 3 倍'

### 人味增强

- [ ] 有第一人称态度表达（'我觉得'、'我发现'、'说实话'）
  - ❌ '该工具具有明显优势'
  - ✅ '我用了一周，确实比之前顺手'

- [ ] 有具体时间感（'用了一周'、'试了 3 天'、'昨天发现'）
- [ ] 技术术语有人话翻译（`[技术词]（[大白话解释]）` 格式）

## 第四遍：细节打磨

- [ ] 句长达标：参考 15-25 字，超过 35 字的句子在逗号处断开
- [ ] 段落达标：手机端每段不超过 5 行，每段 1-2 个观点
- [ ] 朗读顺畅：大声朗读无拗口，停顿自然
- [ ] 标点自然：多用句号，少用逗号连长句；感叹号适度
- [ ] 标题层级清晰，重点内容突出，表格对齐整齐
- [ ] 审校报告在对话窗口展示，**禁止**写入最终 .md 稿件文件

---

## 二次扫描（只扫本轮改动）

- [ ] 本轮改动没有新增假深刻结构
- [ ] 本轮改动没有新增意义拔高和万能展望
- [ ] 本轮改动没有新增金句、排比、漂亮比喻
- [ ] 本轮改动没有新增未经核验的数据、场景、例子
- [ ] 本轮改动没有删掉作者有辨识度的语气和手迹
- [ ] 输出了关键改动说明表：原文 / 修改 / 原因 / 可还原

---

## 图卡模式审校（card_post 模式输出时额外执行）

### 字数与数量

- [ ] 图卡内容排版（封面 + 内容卡正文）总字数不超过 1000 字
- [ ] 卡片总数不超过 9 张（含封面）
- [ ] 每张内容卡只有一个核心主题，标题具体（无空泛口号）
- [ ] 每张内容卡要点数量为 3-4 条

### 事实准确性

- [ ] 所有版本号、发布时间、性能数据、功能名有官方来源
- [ ] `excluded_claims[]` 中的事实未出现在发布内容（【发布配文】和【图卡内容排版】）
- [ ] 未使用「100%」「史诗级」「全网最强」「首个」「唯一」「完全免费」「无限」等绝对化表述
- [ ] 语言有冲击力，但未牺牲事实准确性

### 内容质量

- [ ] 卡片之间没有重复观点
- [ ] 封面包含强钩子标题 + 具体利益点
- [ ] 要点每条不超过 24 个汉字

### 输出格式

- [ ] 输出包含【事实核查摘要】
- [ ] 输出包含【卡片数量决策】
- [ ] 输出包含【发布配文】
- [ ] 输出包含【图卡内容排版】
- [ ] Card 编号连续（Card 01、Card 02…）
- [ ] 输出已落盘到 `{workspace}/output/{YYYYMMDD}-card-{slug}.txt`
